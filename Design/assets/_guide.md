# 에셋 문서 제작 가이드

이 폴더(`Design/assets/`)에 외부 에셋 사용 문서를 모으는 규칙. 새 에셋을 도입·정리할 때 이 가이드를 따른다.

## 왜 여기에 모으나
- 에셋 폴더(`Client/Assets/...`)는 Asset Store **재임포트/업데이트 시 덮어써지거나 삭제**될 수 있음 → 우리가 쓴 문서가 유실됨.
- 따라서 **우리가 작성하는 모든 에셋 문서는 에셋 폴더 밖, git으로 관리되는 `Design/assets/`에 둔다.** 에셋 폴더엔 신규 작성하지 않는다.
- 벤더가 에셋과 함께 배포하는 문서(예: Digger의 `Client/Assets/Digger/CLAUDE.md`, `Documentation/`)는 **링크로만 참조**한다(우리가 복제·요약하지 않음).

## 에셋당 파일 2종 (용도 분리)
| 파일 | 용도 | 작성자 | 비고 |
|---|---|---|---|
| `<asset>.md` | **사람용 메모** — 주의사항·간단 사용법·샘플 참고점·직접 써보며 익힌 것 | 사용자 본인 | 빈 스켈레톤으로 시작, 필요 시 Claude에게 추가 요청 |
| `<asset>.claude.md` | **Claude 소비자 문서** — 내가 이 에셋을 써서 작업할 때 즉시 정확히 행동하게 만드는 정보 | Claude + 사용자 공동 | 아래 섹션 템플릿. 본문은 MCP 샘플 조사로 검증하며 채움 |

- 파일명: 소문자 케밥(예: `easy-build-system.md`, `easy-build-system.claude.md`).
- 두 파일과 벤더 문서 링크는 모두 `Design/assets.md`(인덱스 허브)에 등록한다.
- 루트 `CLAUDE.md`에 인덱스 포인터가 한 줄 있어야 Claude가 인덱스의 존재를 안다(자동 로드되는 건 루트 CLAUDE.md뿐).

## Claude 소비자 문서(`*.claude.md`) 섹션 템플릿
**원칙: 소비·검증·함정 중심. 벤더 문서가 다루는 내부 아키텍처/확장 절차/공식 튜토리얼은 재서술하지 않고 링크로 대체.** 분량은 타이트하게(60~120줄). 핵심은 2·3·5·7.

1. **오리엔테이션** — 무엇/변종/**설치 버전**/리포 내 위치 / 공식문서·벤더문서 링크.
2. **우리가 호출하는 공개 API ★** — 우리가 실제로 부르는 클래스·메서드를 시그니처+파일경로로. 내부 API 말고 소비 표면만.
3. **작업 레시피 표 (task → API 호출) ★** — "런타임에 X 하기 → `Y(...)`". 내가 가장 빠르게 행동하는 인덱스.
4. **최소 동작 스니펫** — 가장 흔한 1동작의 복붙 가능한 최소 예제. **샘플 씬에서 실측 검증된 것만.**
5. **씬/프리팹 셋업 요건 ★** — API 동작에 필수인 GameObject·컴포넌트·계층·초기화 순서.
6. **데이터·좌표·단위 규약** — world/local 좌표, 해상도, 영속 데이터 위치, 직렬화 대상. 틀리면 조용히 깨지는 것.
7. **런타임 vs 에디터 제약 · 함정 ★** — 에디터 전용 호출, 스레딩/비동기, 비용, 초기화 순서 의존, 자주 나는 에러→원인.
8. **우리 프로젝트 통합 지점** — 월드생성 코어 등과의 seam(포인터, 상세는 `Design/tech/` 링크).
9. **에셋 맵 / 쓸만한 리소스 위치** — 큐레이션된 표(폴더 → 무엇 → 쓸모/무시). **전체 재귀 트리 덤프 금지.** 1과 2 사이에 둬도 됨(네비게이션 색인).

> 섹션 순서는 가치순으로 배치(오리엔테이션 → 에셋 맵 → API → 레시피 → 스니펫 → 셋업 → 규약 → 함정 → 통합). 고정 규칙 아님, 에셋에 맞게 조정.

## 검증 마커 (필수)
Claude 소비자 문서의 각 항목/섹션에 신뢰도를 표시한다. "어디까지 믿어도 되는지"를 Claude가 알아야 하기 때문.
- `[확인됨: MCP/샘플/테스트]` — 샘플 씬 실측, 실제 실행/테스트로 확인.
- `[확인됨: 벤더문서]` — 벤더 공식 문서 근거.
- `[가정]` — 코드/이름에서 추정. 미검증.

문서 상단에 **설치 버전·확인 날짜**를 적는다(에셋 업데이트 시 stale 판단용).

## 작성 절차
1. **1차(스켈레톤)**: 폴더/파일 생성, 오리엔테이션·에셋 맵을 raw 스캔(`ls`/`find`)으로 초안 작성(`[가정]`), 나머지 섹션은 헤더 + `TODO` 플레이스홀더. 사람용 메모는 빈 헤더만.
2. **2차(본문 채움, Unity 에디터 필요)**: MCP로 샘플 씬을 띄워 컴포넌트·직렬화 설정·실제 호출을 확인하며 API·레시피·셋업·스니펫을 채우고 `[가정]`을 `[확인됨]`으로 승격. 사용자와 함께 검증.
