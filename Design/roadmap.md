# 로드맵 / 프로토타입 범위

## 완성 기준
- **현재 목표 = 프로토타입.** 디테일 완성도가 아니라 **"필요 부분이 되는지" 검증.**
- 대충 그럴싸한 수준으로 파이프라인 end-to-end가 도는지 먼저, 품질/디테일은 후순위.

## 프로토타입 단계
- **P1(현재)** — 월드 기술 슬라이스 + 기초 생동감. (아래 정의)
- **P2** — 인벤토리 등 게임플레이 본격화. (이후)

## P1 정의 (구체 규모)
- **월드**: 작게. **약 2개 존에 걸친 섬 하나 + 나머지는 물.** 섬 위에 **2~3개 바이옴** 분할.
- **바이옴별**: 지형 생성 + 나무/돌 배치 + 동물/적 **1~2종** 스폰.
- **상호작용(발하임 수준 최소)**:
  - 돌/나뭇가지 **주우면 사라짐**(인벤토리는 P2).
  - 동물: 가까이서 때리면 **일정 시간 플레이어로부터 도망**.
- **카메라**: 3인칭/1인칭 전환. **로우폴리** 아트.
- 생존/제작/건축/사육/튜토리얼 등은 **P1 범위 밖.**
- **스트리밍**: P1은 **세이브 없이 매 시작 재생성** → **정적 전체 로드**(작은 섬 존들을 시작 시 한꺼번에). **동적 스트리밍(로드/언로드) + 세이브/로드는 P2.**
- **구조 규율**: 동적 스트리밍을 빼더라도 존을 **일반 그리드 + 육지 마스크**로 다뤄 P2에서 스트리밍/규모 확장이 데이터 변경으로 되게. (→ [tech/world-generation.md](tech/world-generation.md) D1)
- ⚠️ **참고**: P1 월드가 작아(섬 2존) **스트리밍·perf 스트레스는 가볍게만 검증됨.** 본격 perf 검증(R5)·동적 스트리밍·세이브는 P2. 마이크로튜닝은 P1에서 안 함(작은 월드라 결론 안 남).

## 검증할 핵심 위험 (de-risk)
- **R1** 순수 C# 생성 코어 → Unity Terrain 하이트맵 데이터 흐름.
- **R2** Digger가 절차 데이터를 소비(`IVoxelGenerator`) + 런타임 `SetupRuntimeTerrain`.
- **R3** 전역좌표 노이즈로 존 경계 이음매 0 + Digger 타일 경계.
- **R4** 바이옴 블렌딩 텍스처(지표+동굴벽).
- **R5** 저사양에서 스트리밍 성능.
- **R6** 절차 지형 위 **엔티티/오브젝트 배치**(지면 높이 정렬) + **존 수명주기**(로드 시 스폰/언로드 시 정리) + 기초 상호작용 동사.

## 빌드 순서
1. **순수 C# 코어**: MT 시드 → 전역 하이트필드(저옥타브) + 바이옴 필드 2~3종. (R1)
2. **단일 존**: 하이트필드로 Unity Terrain 생성 + 바이옴 텍스처 + Digger 셋업 + 단순 동굴 1개(우선 Modify). (R2, R4)
3. **유한 스트리밍**: 주변 3×3 존 로드/생성, 경계 이음매 확인. 동굴을 전역 SDF 필드로 전환. (R3)
4. **전역 해수면**: Stylized Water 2(시각).
5. **Suriyun 캐릭터**: 보행/동굴 진입/존 횡단 체감 + 저사양 성능. (R5)
6. **생동감/상호작용**: 동물·몬스터 스폰(지면 정렬, 존 수명주기 연동) + 나무·바위 배치 + 기초 채집/타격 동사. (R6)

→ **첫 마일스톤 = 1+2.** R2가 가장 novel한 통합 위험이라 스트리밍 전에 단일 존에서 먼저 깸. 스폰/상호작용(6)은 월드 토대(1~5) 검증 후.

## 마일스톤 1+2 태스크 분해

### M1 — 생성 코어 `BW.WorldGen.Core` (순수 C#)
- **T1.1** asmdef 생성(`Assets/_Core/WorldGen`) + FastNoiseLite 벤더링 + MT RNG 편입.
- **T1.2** Config(plain C#): worldSeed, zoneSize=128, 하이트맵 해상도, seaLevel, 레이어별 노이즈 파라미터.
- **T1.3** 필드 샘플러: Continentalness/Elevation/Temperature/Moisture (전역좌표, 시드 오프셋; Erosion 생략).
- **T1.4** `BiomeClassifier.BiomeAt(worldPos)`: 대륙도→바다/육지, 육지면 고도+온도+습도→바이옴 가중치(연속 블렌딩).
- **T1.5** 높이 합성: 베이스 고도 + 바이옴 가중 셰이핑 → 사각 영역 `HeightField`. (P1 섬=명시적 반경 마스크)
- **T1.6** 검증: 그리드 샘플 → PNG/CSV로 하이트맵·바이옴맵 덤프, 에디터에서 눈으로 확인.

### M2 — 단일 존 `BW.WorldGen.Unity` (에디터 타임)
- **T2.1** asmdef 생성(`Assets/_ProjectBW/Scripts/WorldGen`), Core+Digger 참조.
- **T2.2** `ZoneBuilder` 에디터 메뉴: TerrainData + `SetHeights`(코어 HeightField) → Terrain GO.
- **T2.3** Terrain layers 7장 등록 + `SetAlphamaps`(바이옴 가중치 + 경사 규칙).
- **T2.4** Digger 셋업(템플릿) + 단순 동굴 1개 Modify 카빙.
- **T2.5** 검증: 지형/바이옴 텍스처/동굴 확인 + 캐릭터 보행(Suriyun/FlyCamera 재사용).

→ 착수 순서: T1.1 → … → T1.6(코어 눈으로 확인) → T2.1 → … → T2.5.

## 프로토타입 잠정 파라미터
- **존 크기**: **한 변 128m**(시야거리 보고 조정).
- **스트리밍 반경**: 3×3. (P1 섬이 작아 실제로는 거의 전부 로드)
- **바이옴**: 정의는 4종(초원/설산/사막/암석지대), **P1은 섬에 2~3종만 실제 배치**.
- **텍스처 레이어**: 7장(URP 8장 내, MicroSplat 보류). 상세 → [tech/world-generation.md](tech/world-generation.md).
