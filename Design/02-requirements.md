# 02. 요구사항

## 기능 요구 (상위)
- 랜덤 시드 기반 절차적 지형 생성.
- 생성 지형에 Digger 적용(동굴/지형 변형).
- 절차적 동굴 생성.
- 물/바다 시스템(단계적).
- 마인크래프트식 바이옴 정보 → 기본 바닥 텍스처 등 변경.

## 비기능 요구 (NFR) — 전 결정에 가중치
- **저사양 타깃**: 옵션 타협을 해서라도 저사양에서 구동. 모든 기술 선택에 **연산/메모리 부하 최소화** 가중치.
- **1회성 비용 허용**: 최초 맵 생성에 다소 시간 걸리는 건 1회성이라 허용 → 무거운 처리는 **생성 시점(첫 방문)**으로 몰고 런타임 상시 부하 최소화.
- **결정성(determinism)**: 시드+버전 동일 → 동일 월드. 시드 재생성·서버/클라 합의의 전제. (메르센 트위스터 C# 직접 구현 완료)
- **아트가 부하를 낮춤**: 툰/SD → 비싼 PBR/리얼 라이팅 불필요. 물도 스타일라이즈드.

## 타깃 플랫폼 / 서버 방침
- **현재 완성 기준 = 프로토타입** (싱글 클라이언트, Unity).
- **멀티플레이 = 싱글 + 소규모 코옵.** 최대 동접 **~16명**. 데디케이티드 서버로 지속 월드 공유. **MMO 아님.**
  - (MMO는 네트워크 자체는 가능하나 **Digger식 복셀 지형을 포기해야** 해서 제외.)
- **데디케이티드 서버 지원 예정** — 단 **프로토타입에선 보류**(싱글부터). 상세 → [tech/server-networking.md](tech/server-networking.md).
- 환경: Unity 6000.4 / URP 17.4.

## 성능 / 하드웨어 타깃 (PC 전용)
- **PC만 타깃** (콘솔/모바일 제외).
- **희망**: Intel 8세대 노트북(i5-8250U + 8GB + 내장그래픽) 저해상도 30fps.
- **현실 하한 목표**: Ryzen Renoir(4300U) 또는 Intel 11세대(Iris Xe) + 8GB + 내장그래픽 — 옵션 다 끄면 플레이 가능 수준.
- **데스크탑 마지노선**: GeForce GTX 1050 급.
- 아트: **로우폴리** 캐릭터/오브젝트 + 툰. "옵션 최소 = 구동 가능"이면 충족.
- **개발/테스트 머신**: i5-5200U(8/12GB), i5-8250U(16GB), i7-1280P(32GB), Ryzen AI Pro 7 350(64GB), Mac Studio M1 Max, eGPU GeForce 5060. (i5-2540M은 DX10 한계로 논외)

## 결과적 선호
스트리밍 반경 작게 · Digger 청크 해상도 보수적 · 동굴 밀도 절제 · 노이즈 단순/저옥타브 · 바이옴 블렌딩 비용은 생성 시점 1회로.
